Object methods, "this": https://javascript.info/object-methods

The value of "this" is the object “before dot”, the one used to call the method.

Remember: A function that is a property of an object is called its method.

In JavaScript, keyword this behaves unlike most other programming languages. It can be used in any function, even if it’s not a method of an object.

Arrow functions are special: they don’t have their “own” this. If we reference this from such a function, it’s taken from the outer “normal” function.

This in JavaScript: https://zellwk.com/blog/this/

What is this? It is a keywork whose value changes depending how a function gets called.

    There are 6 different ways where "this" can take on new values:
  • in global context - usually you wouldn't use "this" but the value of this defaults to the "window" object.
  • in object construction - "this" refers to the instance.
  • in an object method - "this" within any method refers to the object iteself.
  • in a simple function - "this" is always set to "window", to fix you can make a variable to store the reference to "this" or use an arrow function.
  • in an arrow function - in an arrow function is always the same as "this" around it.
  • in an event listener - "this" is set to the element that fired the event in an event listener.

"this" is a crucial keyword in JavaScript. It appears in many Javascript frameworks. Some will say don't use "this" because it's confusing but it is important to be able to understand and use "this". Articles like this one can be used as clarification.

Novice to Ninja Chapter 5 - Objects

Think of an object as a dictionary where you look up a property name and see a value.

const spiderman = {}; - is an example of creating an object literal.

You can access properties of an objects properties by using dot notation or bracket notation.

New properties and methods can be added to objects at any time in a program. You just assign a value to the new property and it will be slotted under the object. Similarly you can change the value of an object's properties at any time.

To remove properties from objects you can use the "delete" operator.
For Example:
delete superman.fly
^ This removes the fly property of the object superman.

Objects are not ordered lists like an array or set map. They can change order.

(Once again!) The keyword "this" refers to the object that it is within. It can be used inside methods to gain acess to the object's properties.

Namespacing can be used to prevent duplication of functions. This duplication can happen if multiple developers are working in a library or a large sample of code is worked on for a prolonged time, so namespacing can be helpful. Creating a namespace is just making an object that holds the function or functions.

JSON (JavaScript Object Notation) is very helpful in being human and machine readable.

Novice to Ninja Chapter 6 - Document Object Model

The DOM allows you to access elements of a web page and enable interation.The textbook defines this as it represents an HTML document as a network of connected nodes that form a tree-like structure. Everything is a node!

The DOM is language agnostic although it is most commonly used with Javascript.

    DOM Methods- most of these do as their name suggests
  • getElementById()
  • getElementsByTagName()
  • getElementByClassName()
  • querySelector() - allows you to use the CSS notation to find the first element in the document that matches. Note that suedoselectors can be used example: 'li:last-child'
  • querySelectorAll() - allows you to use the CSS notation to find all the elements in the document that match.
    Navigating the DOM tree with properties:
  • childNodes
  • children
  • firstChild
  • lastChild
  • parentNode
  • nextSibling
  • previousSibling

    • These methods find the node not the element. Using these properties allows you to navigate around the whold document tree.

Various different getter and setter methods can be used to reference attributes. For example: getAttribute() and setAttribute()

Novice to Ninja Chapter 7 - Events

Events are another part of the DOM and they are what provides the link between the web page and user interactions. Clicking on a link, pressing a key or moving a mouse can be detected and responded to.

Think of event listeners as alarms of sorts. They notify us of something happening so we can configure what happens once that something happens.

Old ways to avoid:

Inline event handlers used to be the way to go (think of the onclick="" attribute). This however isn't recommended. Another older way is to use event handler properties that all node objects have (document.onclick for example).

Use Event Listeners!

The current and recommended standard is to use Event Listeners. They are supported in all modern browsers, separate HTML form JS and help with organization.

    addEventListener() Types of Events
  • Mouse Events
  • click
  • mousedown
  • mouseup
  • dblclick

    • Remember avoid using click and doubleclick event on the same element.

  • mouseover
  • mouseout
  • mousemove
  • Keyboard Events
  • keydown
  • keypress
  • keyup

    • Properties included: shiftKey, ctrlKey, altKey, metaKey

  • Touch Events
  • touchstart
  • click
  • touchend
  • touchmove
  • touchenter
  • touchleave
  • touchcancel

    • Properties included: touches and length

An event listener can be removed using the removeEventListener() method.
